## Known gotchas
- `openssl-sys` and `pq-sys` need PKG_CONFIG_PATH and OPENSSL_DIR set — without them, linker errors on `lssl`/`lpq`.
- 215 lib tests are ignored — these are postgres DB tests (require `docker-compose up` to run a local Postgres). They are correctly skipped with `--lib` flag.
- Toolchain file is `rust-toolchain` (no `.toml` extension) — rustup still respects it.
- The `contracts/` directory is excluded from the workspace in `Cargo.toml` (`exclude = ["contracts/"]`), but individual contract crates under `contracts/generated/` are pulled in as path dependencies and compile fine.
- `e2e` crate is in the workspace (tests require Anvil + Postgres). Its lib code compiles clean; integration tests are out of scope for `--lib` runs.
- Upstream uses `cargo-nextest` in CI; `cargo test` works for lib tests but may behave slightly differently for tests using global state.
