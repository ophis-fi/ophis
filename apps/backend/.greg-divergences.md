# apps/backend — divergences from upstream cowprotocol/services

Files removed or modified beyond the original subtree merge. When running
`git subtree pull --prefix=apps/backend services-upstream main --squash`,
expect these to come back as untracked additions; re-apply the deletions
listed below.

Upstream ships Claude-Code-specific tooling that's inert in our repo
(workflow files in subdirectories don't trigger CI; MCP servers without
their env file can't start). Removed to keep our tree free of unused

- `apps/backend/.mcp.json`

- `apps/backend/.github/workflows/claude-code-review.yml`

## Modified (Ophis-only, 2026-05-17): chain 999 in `contracts/`

Added `HYPEREVM = 999` constant + three entries in the contracts crate's
per-network registry so the baseline solver can resolve UniswapV3
helpers on HyperEVM mainnet:

- `contracts/src/networks.rs` — added `pub const HYPEREVM: u64 = 999;`
- `contracts/src/main.rs` — added `HYPEREVM => "..."` entries to:
  - `UniswapV3QuoterV2.with_networks(...)`  → `0x03a918028f22d9e1473b7959c927ad7425a45c7c` (HyperSwap V3 QuoterV2)
  - `UniswapV3SwapRouterV2.with_networks(...)` → `0x6D99e7f6747af2cdbb5164b6dd50e40d4fde1e77` (HyperSwap V3 SwapRouter02)
  - `IUniswapV3Factory.with_networks(...)` → `0xB1c0fa0B789320044A6F623cFe5eBda9562602E3` (HyperSwap V3 Factory)
- `contracts/generated/contracts-generated/{uniswapv3quoterv2,uniswapv3swaprouterv2,iuniswapv3factory}/src/lib.rs` — regenerated by `cargo run -p contracts-generate` (the codegen reads `main.rs` + writes per-network lookup tables into the facade crates).

**Why these need to be here, not in upstream:** the upstream
`cowprotocol/services` doesn't ship for HL. The original audit on PR #50
(2026-05-16) caught that without these registrations the
`solvers/src/boundary/baseline.rs:233-236` quoter-resolution returns
`None`, which silently drops every `Liquidity::Concentrated(pool)` —
making the entire `[[liquidity.uniswap-v3]]` block dead plumbing.

**Project X intentionally NOT registered.** Both Project X's SwapRouter
(`0x1EbDFC75…Af9B`, V1 — 8-tuple `ExactOutputSingleParams` with `deadline`)
and Quoter (`0x239f11a7…5258`, V1) use the V1 ABI. The driver encodes V2
calldata, so a V1 router would revert every settlement at simulation.
Adding V1 router support is a follow-up spec, not in this divergence.

**Conflict-recovery on `git subtree pull`:** upstream may add other
chains to these three registries. Merge their additions, keep our
`HYPEREVM =>` lines. After merge: `cd apps/backend/contracts && cargo run`
to regenerate the facade crates; commit alongside the subtree pull.

## Conflict-recovery procedure

When `git subtree pull` re-introduces any of the files above:

1. `git rm` the files listed in the "Removed" section.
2. Stage + commit alongside the subtree pull.
3. Update the date of this section so the next operator knows the
   re-purge was applied.
