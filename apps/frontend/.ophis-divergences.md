## Removed (CoW affiliate subsystem, 2026-06-11)

The inherited CoW affiliate UI was removed in two PRs (inventory below). Upstream
still ships these files, so a future `git subtree pull` re-introduces them; the
**conflict-recovery procedure** at the end of this file re-deletes them. Ophis runs
its OWN native affiliate / rebate system (`modules/affiliate/lib/ophisAffiliateApi.ts`
-> `rebates.ophis.fi`, EIP-191), which is NOT upstream and must be preserved.

- **#550** removed the CoW "partner" affiliate page (EIP-712 code-mint that POSTed
  to CoW's `bff.barn.cow.fi`): `pages/Account/AffiliatePartner.tsx` + its
  containers/hooks/pure components + the partner bff methods.
- **#558** removed the CoW "Trader" affiliate subsystem (75 files: the My Rewards
  page, `AffiliateTrader*` containers/pure, ~14 trader hooks, `api/bffAffiliateApi*`,
  trader atoms, the RewardsRow render sites in 6 trade widgets). It severed every
  `bff.barn.cow.fi/affiliate/*` call. CRITICAL edit to KEEP on re-merge:
  `modules/appData/updater/AppDataUpdater.tsx` must NOT call `useIsRefCodeExpired()`
  (it leaked the connected wallet to CoW on the trade path); pass `refCode` straight
  through from `affiliateTraderSavedCodeAtom`.

KEEP on every subtree pull: the native `ophisAffiliateApi` block, `OphisAffiliateDashboard`,
`useOphisAffiliateSign`, `RefCodeCaptureUpdater`, `refBoundWalletsAtom`,
`affiliateTraderSavedCodeAtom`, `useAffiliateTraderCodeFromUrl`, the `/profile`
`/leaderboard` `/partner` routes, and `libs/common-const/src/bff.ts` `BFF_BASE_URL`
(still used by the bridging socket proxy + price/balance endpoints, unrelated to affiliate).

## Conflict-recovery procedure

After `git subtree pull --prefix=apps/frontend cowswap-upstream main --squash`:

1. **Modified / Added files** (the sections above): for each conflict, inspect the
   upstream change with `git log` on the file, then reapply Ophis's intent on top of
   it, preserving both.
2. **Removed files** (the `## Removed (CoW affiliate subsystem)` section above):
   upstream re-adds the deleted CoW affiliate files (they reappear as additions, not
   conflicts), so re-apply the cleanup. The two removal commits are the AUTHORITATIVE
   lists; do not rely on the prose globs alone (they undercount). Run from repo root:
   - Files to RE-DELETE (`git rm` each):
     `git show 2fb41475b 1252eb784 --diff-filter=D --name-only --format= -- apps/frontend`
     (14 from #550 = the CoW partner page + its helpers; 75 from #558 = the Trader
     subsystem). It spans `pages/Account/Affiliate{Partner,Trader}.tsx` and the CoW-only
     `modules/affiliate/**` files: `api/bffAffiliateApi*` + `api/api-utils.ts` +
     `api/checkIfTraderHasPastTrades*`, the `Affiliate{Partner,Trader}*` containers, the
     trader hooks, the `pure/` trader components (`AffiliateTrader*`, `AffiliateTraderModal`,
     `RefCodeInput`, `shared`, `Donut*`, `HowItWorks`, `MetricsCard`, `NextPayoutCard`,
     `TraderIneligible`, `UnsupportedNetwork`), the trader state atoms, `README.md`, and
     `misc/*.sql` (none of which the globs match).
   - Files to RE-EDIT (remove the dead wiring):
     `git show 2fb41475b 1252eb784 --diff-filter=M --name-only --format= -- apps/frontend`.
     The wiring set: `common/constants/routes.ts`,
     `legacy/components/Header/AccountElement/index.tsx`, `modules/affiliate/index.ts`
     (barrel), `modules/application/containers/App/RoutesApp.tsx`, `.../App/menuConsts.tsx`
     (+ `menuConsts.test.ts`), `.../AppContainer/AppContainer.container.tsx`,
     `.../AppMenu/index.tsx`, `pages/Account/Menu.tsx`, `pages/Account/index.tsx`,
     `libs/common-const/src/common.ts` (drop `PAGE_TITLES.MY_REWARDS` / `AFFILIATE`), and
     the 6 `RewardsRow` trade-widget render sites. Typecheck (step 4) catches dangling
     imports but NOT dead string links (e.g. a `/account/my-rewards` menu entry survives
     typecheck), so replay these by hand.
   - Re-apply the CRITICAL edit: `modules/appData/updater/AppDataUpdater.tsx` must NOT
     call `useIsRefCodeExpired()` (the `bff.barn.cow.fi` wallet leak on the trade path);
     pass `refCode` straight through from `affiliateTraderSavedCodeAtom`.
   - PRESERVE the native Ophis system + `BFF_BASE_URL` (KEEP list above). Note: most KEEP
     items are themselves Ophis divergences that must be RE-APPLIED if upstream overwrote
     their surrounding files, not merely left alone; `affiliateTraderSavedCodeAtom` and
     `useAffiliateTraderCodeFromUrl` are upstream-origin files Ophis intentionally reuses.
   - Confirm severance: `grep -rE "bffAffiliateApi|bff\.barn\.cow\.fi/affiliate" apps/frontend/apps/cowswap-frontend/src`
     returns zero hits (the only remaining `bff.barn.cow.fi` is the bridging socket proxy).
3. Update this file's `Phase X, YYYY-MM-DD` annotations for anything re-applied.
4. `npx nx typecheck cowswap-frontend` (catches any dangling affiliate import), then
   re-run the local Phase 1 forked-Gnosis stack (`docker compose -f infra/local/docker-compose.fork.yml --env-file infra/local/.env up`) and the Phase 1.5 Sepolia smoke test (Task 5 of Phase 1.5 plan) to confirm the fork still settles and partner-fee still injects.
